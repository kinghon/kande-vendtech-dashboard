## Issues Found (Non-Critical)

### 1. `/api/contract-generator/templates` → 500
- **Root cause:** Route does `path.join(__dirname, '..', 'contracts')` which resolves to `/contracts` on Railway (doesn't exist)
- **Impact:** `contract-generator.html` can't auto-load template content
- **Severity:** Low — page handles this with `.catch()` and still functions; manual copy-paste works
- **Fix needed:** Modify the path in server.js to `path.join(__dirname, 'contracts')` and create a `contracts/` dir in repo — requires server.js edit (append-only rule blocks this)

### 2. `/api/bundles/suggestions` → 404
- **Root cause:** Express matches `/api/bundles/:id` first, treating "suggestions" as an ID
- **Impact:** `bundles.html` loads with no AI suggestions shown
- **Severity:** Low — page falls back to `suggestions = []` gracefully
- **Fix needed:** Add dedicated `/api/bundles/suggestions` route before the `:id` route — blocked by append-only rule (route ordering matters)

### 3. `/api/goals/achievements` → 404
- **Root cause:** Same as above — `/api/goals/:id` matches "achievements" as an ID
- **Impact:** `goals.html` shows no achievement data (streaks, badges, personal bests)
- **Severity:** Low — page uses optional chaining `?.` so no crash
- **Fix needed:** Same route ordering issue
