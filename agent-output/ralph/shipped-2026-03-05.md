## Bug Fixed

**machines.html — Machine card showed "undefined" as name**

**Root cause:** The only machine in the DB was created with a `model` field (`"test"`) but no `name` field. The UI was rendering `m.name` directly with no fallback, resulting in "undefined" displayed on the card and in the detail modal.

**Fix:** Added fallback chain `m.name || m.model || 'Machine #' + m.id` in two places:
1. Machine card title (line ~553)
2. Detail panel title (line ~641)

**Commit:** `09bd933` — `[ralph] fix: machines page shows 'undefined' for machine name — fallback to model or id`
