### `/api/contract-generator/templates` — 500 → 200 ✅

**Root cause:** The endpoint read contract templates from `path.join(__dirname, '..', 'contracts')` — a parent directory that doesn't exist on Railway's deployment environment. Only worked locally because `/Users/kurtishon/clawd/contracts` was a symlink to `pb-info-src/kande-vendtech/contracts/`, and symlinks outside a git repo don't deploy.

**Fix:**
1. Copied 6 contract template `.md` files into `kande-vendtech/contracts/` (now tracked in the repo)
2. Appended a route-patch to `server.js` (APPEND-ONLY) that uses `app._router.stack` to splice out the broken handler and register a corrected one pointing to `__dirname/contracts`
3. Merged with remote conflict (another ralph commit had landed on origin/main), rebased cleanly
4. Pushed commit `524e02a` → deployed, verified 200
