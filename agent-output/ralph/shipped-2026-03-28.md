## Bug Fixed

**`/api/analytics/property-types` — Recommendation was always "Not enough data yet"**

- **Root cause:** The recommendation logic only triggered when `conversion_rate > 0` (i.e., at least one signed deal). With 0 signed deals across 760 prospects, the message was always the placeholder — useless.
- **Fix:** Added two intermediate branches:
  1. If there are hot leads → show which property type has the most hot leads + largest pipeline type
  2. If no hot leads but prospects exist → show largest pipeline + highest engagement type
  3. Only falls back to placeholder if there are literally no prospects at all
- **Before:** `"Not enough data yet. Keep adding prospects to see trends."`
- **After:** `"No signed deals yet — focus energy on Apartment (22 hot leads, highest urgency). Healthcare has the largest pipeline (124 active prospects)."`
