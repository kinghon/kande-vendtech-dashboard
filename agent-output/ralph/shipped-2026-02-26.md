## Bug Fixed: `pipelineCards.filter is not a function`

**File:** `home.html` (line 1162)

**Symptom:** Console error on every dashboard load:
```
Dashboard load error: TypeError: pipelineCards.filter is not a function
```

**Root cause:** `/api/pipeline/cards` returns `{cards: {new_lead: [...], contacted: [...], ...}, stages: [...], total: N}` — a stage-keyed dict — but `home.html` assigned `results[4]` directly to `pipelineCards` expecting a flat array. `.filter()` on a plain object throws.

**Fix:** Added response shape detection that flattens the stage-keyed dict into a flat array. Now correctly reads 534 pipeline cards.

**Commit:** `36f1113` — `[ralph] fix: flatten pipelineCards from API dict to array in home.html`
