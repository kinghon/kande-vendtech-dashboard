### Bug Fixed: usage.html timezone offset
Problem: After 4pm PT (midnight UTC), "Turns Today" showed 0 and "Est. Daily Cost" showed $0.00.
Root cause: todayStr() and last7DaySlots() used new Date().toISOString() (UTC). At 8pm PT (4am UTC Feb 28),
today became 2026-02-28 but all slot data was dated 2026-02-27 — zero matches.

Fix: Updated both functions to use Pacific time via:
  toLocaleDateString('en-CA', { timeZone: 'America/Los_Angeles' })
