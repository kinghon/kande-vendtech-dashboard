## Fix: Property Type Normalization

**Problem:** 20 recently-added prospects (added ~Mar 29–Apr 1) had non-canonical property_type values from new imports using mixed-case and slash-separated formats.

**Root cause:** New prospects added via Apollo/scout/relay imports after the last normalization run (Mar 29) weren't passed through the normalizer.

**Fixed via:** POST /api/prospects/normalize-types

**Changes (20 records):**
- `Office/Business` → `office` (10 records: ids 7847, 7855, 7860, 7881, 7886, 7894, 7900, 7971, 7977, 7995, 8002, 8017)
- `medical` → `healthcare` (3 records: ids 7930, 7932, 7936)
- `manufacturing` → `industrial` (2 records: ids 8029, 8031)
- `Apartments/Condos` → `apartment` (1 record: id 7869)
- `Healthcare` (mixed-case) → `healthcare` (1 record: id 8007)
- `Senior Living` (mixed-case) → `senior_living` (1 record: id 7840)

**Result:** Property type analytics now shows clean canonical types only.
