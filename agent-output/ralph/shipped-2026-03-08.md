## Bug Fixed

**Pipeline Board — Recent Activity sidebar showed "Unknown" for all prospect names**

**Root cause:** `loadRecentActivity()` fetched `/api/activities?limit=20`. The `/api/activities` route returns raw DB records — these have `prospect_id` but no `prospect_name` field. The template used `a.prospect_name || 'Unknown'`, which always fell through to 'Unknown'.

**Fix:**
1. Added `/api/activities/enriched` endpoint to `server.js` (append-only rule respected). It joins each activity with `db.prospects` to attach `prospect_name` before returning.
2. Updated `pipeline-board.html` to call `/api/activities/enriched?limit=20` instead of the bare activities endpoint.

**Commit:** `d9afca2` — `[ralph] fix: pipeline board Recent Activity shows 'Unknown' — join prospect_name via new /api/activities/enriched endpoint`
