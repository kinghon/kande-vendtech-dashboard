## Bug Fixed: Memory Page — File Names Invisible (White Text on Light Theme)

**File:** `memory.html` (line 142)

**Symptom:** On the `/memory` page, the left file list panel showed only file sizes and timestamps (e.g., "20 KB • 2026-03-13T23:04:08Z") with NO file names visible above them. Users couldn't tell what any file was — "AGENTS", "KURTIS-CONTEXT", daily log dates, etc. were all invisible.

**Root cause:** `.file-name` CSS had `color: #fff` hardcoded — a leftover from when this page used a dark theme. The light theme uses white/light-gray card backgrounds (`#ffffff`, `#f1f5f9`), making white text completely invisible.

**Fix:** Changed `color: #fff` to `color: var(--text)` which resolves to `#1a202c` (dark text) in the light theme.

**Commit:** `8900903` — `[ralph] fix: memory page file names invisible — white text on light theme background`
