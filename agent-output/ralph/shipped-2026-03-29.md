## Fix Shipped: Property Type Normalization

**Problem:** `property_type` field had duplicate canonical types due to mixed-case data from imports and legacy records:
- "Apartment" / "apartment" / "Apartments/Condos" → all meant the same thing
- "Office" / "office" / "Office/Business"
- "Healthcare" / "healthcare" / "medical"
- "Senior Living" / "senior_living" / "Senior Living"
- ...etc

This caused analytics endpoints to see "Apartment" and "apartment" as separate buckets, inflating category counts and splitting pipeline analysis.

**Fix:**
1. Added `POST /api/prospects/normalize-types` endpoint to server.js  
   - Maps all variant spellings → canonical lowercase values  
   - Supports `dry_run: true` for preview  
2. Ran the endpoint: **338/769 records normalized**
