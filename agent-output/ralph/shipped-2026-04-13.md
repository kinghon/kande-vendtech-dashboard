## P1: Agent Model Monitor Panel — SHIPPED ✅

### Problem
DeepSeek silently fell back to Opus for days, costing $60+/day undetected. No visibility into which model each cron agent is actually running.

### Solution
Built a full model monitor pipeline:

**1. API Endpoints** (server.js)
- `POST /api/agents/model-sync` — accepts pushed model+cost data from Mac mini
- `GET /api/agents/model-status` — returns per-agent: job name, model, tier, cost/24h, last run
- Both on public paths (no auth needed for sync/data reads)

**2. Push Script** (`scripts/push-model-status.py`)
- Scans `~/.openclaw/agents/main/sessions/*.jsonl` for last 24h
- Extracts `model_change` events for actual model used
- Extracts `usage.cost.total` for cost tracking
- Parses cron job name from first user message `[cron:UUID job-name]`
- Classifies into tiers: 🟢 free (DeepSeek/local), 🟡 cheap (Sonnet), 🔴 expensive (Opus)
- Pushes to VendTech API

**3. Office Page UI** (office.html)
- New "🤖 Agent Model Monitor" panel between cost banner and stats
- Table columns: Agent | Model | Cost/24h | Tier | Last Run
- Rows color-coded by tier (green bg = free, yellow bg = cheap, red bg = expensive)
- Auto-refreshes every 2 min
- Sync age indicator + stale warning

**4. Integrated into sync script** (sync-mission-control.sh)
- Runs push-model-status.py every sync cycle
