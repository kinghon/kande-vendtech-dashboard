# Jumpgate LinkedIn — December 8, 2026

**Topic:** AI agent failure detection — what to monitor after month one

---

Our monitoring agent went silent for six days.

Nobody noticed.

Not because we were careless. Because every surface-level metric said everything was fine. Process running. No error logs. No crashes.

The problem: the agent was producing 0 bytes of useful output. And nothing was checking for that.

After 14 months of running production AI agents, here's what we actually monitor:

**1. Output volume** — not uptime. An agent can be "running" while doing nothing.

**2. Output size** — sessions completing with <1KB output means the agent is alive but broken.

**3. Handoff timing** — in a multi-agent system, delays between agents compound fast.

**4. Repetition patterns** — an agent stuck in a loop looks productive. It isn't.

**5. Error impact, not error rate** — a 2% error rate in a critical path is worse than 10% in a non-critical one.

The hardest shift: stop monitoring the agent, start monitoring what it produces.

Uptime is necessary. It's not sufficient.
