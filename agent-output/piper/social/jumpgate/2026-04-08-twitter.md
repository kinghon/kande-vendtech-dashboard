# Jumpgate Twitter — April 8, 2026

## Topic: Multi-agent orchestration — the framework trap

---

Three "conductor vs swarm" multi-agent articles dropped this week.

All of them focus on coordination patterns.

None mention the actual hard problem: what happens when Agent A's output changes what Agent B should do — and neither agent knows it.

After 14 months in production, our "orchestration framework" is:
- Shared markdown files
- A naming convention
- Cron jobs

It survived a 6-day agent failure. LangGraph routing configs don't survive a major version bump.

Invest in decisions that are framework-independent. The coordination pattern is the fifth most important choice you'll make.
