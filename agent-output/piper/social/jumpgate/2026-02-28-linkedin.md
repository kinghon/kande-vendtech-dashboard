What does an AI agent team actually look like in production?

Not in a demo. Not in a pitch deck. In a real business, with real sales targets and real consequences when something breaks.

We've been running one for the past six weeks — five agents, each with a defined job:

**Scout** does prospect research. Every morning, new leads — pre-qualified, scored, and deduped against 300+ existing records. It's now mapped the entire Las Vegas/Henderson vending market (463 prospects across all verticals) and switched itself to quarterly monitoring mode because broad daily scans stopped returning new results. That's the correct behavior.

**Relay** handles email outreach. But it's not sending templates — it's generating personalized openers based on Scout's notes, tracking external vs internal email opens to distinguish real interest from noise, and flagging when a lead has hit email saturation and needs a phone call instead. This week: 77.2% open rate, 15.8% reply rate.

**Ralph** builds and maintains infrastructure. CRM dashboards, API endpoints, call coaching systems, monitoring alerts. When an agent goes offline, Ralph's monitoring catches it. We learned the hard way what happens when it doesn't.

**Piper** handles content — blog posts, LinkedIn drafts, email copy — based on what the rest of the team is learning in the market.

And there's a fifth agent handling a completely separate business line from the same infrastructure.

The system isn't magic. We got things wrong. We built too many dashboards and not enough execution tools. We had a critical agent go offline for 6 days without anyone noticing, during peak season.

But the learning compounds fast when it's written down in shared memory files instead of trapped in one person's head.

We wrote up the full story — how each agent works, the mistakes we made, what we'd do differently. Link in the first comment.

If you're trying to figure out what AI agents could actually do for your operations: this is a working example.
